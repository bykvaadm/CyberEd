# Практические лабораторные по InfraSec и DevOps

Учебный репозиторий с набором практических лабораторных работ для изучения DevOps-инструментов, контейнеризации и
безопасности инфраструктуры.

---

## 📚 Содержание

В репозитории представлено **8 лабораторных работ**, каждая из которых посвящена отдельному этапу построения и защиты
инфраструктуры:

| № | Тема                  | Описание                                                  |
|---|-----------------------|-----------------------------------------------------------|
| 1 | Виртуальная машина    | Разворачивание и базовая настройка ВМ с помощью terraform |
| 2 | Ansible Hardening     | Автоматизация настройки безопасности                      |
| 3 | SSL + NGINX           | Настройка HTTPS и авторизация по сертификатам             |
| 4 | Docker                | Сборка собственного контейнера                            |
| 5 | Побег из контейнера   | Практика container escape                                 |
| 6 | Kubernetes            | Развертывание кластера                                    |
| 7 | Kubernetes Goat       | Практика безопасности Kubernetes                          |
| 8 | Vault + Jenkins в K8s | CI/CD с получением секретов из Vault                      |

---

# 🧪 Лабораторные работы

---

## Лабораторная 1 — Поднимаем виртуальную машину

**Цель:**  
Развернуть виртуальную машину и подготовить локальный terraform для всех остальных лабораторных

**Задачи:**

**Результат:**  
Рабочая ВМ с доступом по SSH и установленным docker

---

## Лабораторная 2 — Hardening с помощью Ansible

**Цель:**  
Научиться автоматизировать настройку безопасности сервера.

**Изучаем:**

- Структуру Ansible-проекта
- Inventory
- Playbooks
- Роли
- Базовый Linux hardening

**Примеры задач:**

- Отключение root login
- Настройка SSH
- Настройка firewall
- Обновление пакетов
- Минимизация установленных сервисов

**Результат:**  
Автоматизированная настройка безопасного сервера.

---

## Лабораторная 3 — SSL в NGINX + авторизация по сертификатам

**Цель:**  
Настроить HTTPS и mutual TLS.

**Изучаем:**

- Генерацию CA
- Выпуск серверного сертификата
- Выпуск клиентских сертификатов
- Настройку HTTPS в NGINX
- Авторизацию по клиентскому сертификату

**Результат:**  
NGINX с включённым SSL и доступом только по валидному клиентскому сертификату.

---

## Лабораторная 4 — Сборка Docker-контейнера

**Цель:**  
Освоить создание Docker-образов.

**Изучаем:**

- Dockerfile
- Слои образа
- Best practices
- Минимизацию образа
- Запуск и тестирование контейнера

**Результат:**  
Собранный и запущенный Docker-контейнер.

---

## Лабораторная 5 — Побег из контейнера

**Цель:**  
Понять ограничения контейнерной изоляции.

**Изучаем:**

- Namespace
- Capabilities
- Привилегированные контейнеры
- Уязвимые конфигурации Docker

**Практика:**

- Анализ конфигурации
- Попытка выхода за пределы контейнера
- Разбор причин уязвимости

**Результат:**  
Понимание рисков неправильной конфигурации контейнеров.

---

## Лабораторная 6 — Поднимаем Kubernetes

**Цель:**  
Развернуть Kubernetes-кластер.

**Изучаем:**

- Pods
- Deployments
- Services
- ConfigMaps
- Secrets

**Результат:**  
Рабочий Kubernetes-кластер.

---

## Лабораторная 7 — Kubernetes Goat

Используется учебный проект:  
https://github.com/madhuakula/kubernetes-goat

**Цель:**  
Практическое изучение уязвимостей Kubernetes.

**Изучаем:**

- RBAC misconfiguration
- Privilege escalation
- Container breakout
- Network policies
- ServiceAccount токены

**Результат:**  
Практический опыт эксплуатации типичных ошибок безопасности Kubernetes.

---

## Лабораторная 8 — Vault и Jenkins в Kubernetes

**Цель:**  
Настроить безопасный CI/CD-процесс с использованием Vault.

**Задачи:**

- Установить HashiCorp Vault в Kubernetes
- Установить Jenkins
- Настроить интеграцию Jenkins ↔ Vault
- Создать пример pipeline
- Получить секрет из Vault во время выполнения пайплайна

**Результат:**  
Рабочий CI/CD pipeline с безопасным хранением и использованием секретов.

# 🛠 Требования

Базовые:

- Linux / macOS. (Под windows автоматизация не заточена, хотя особо упорные студенты запускают в WSL, наверное, хз)
- git
- terraform
- ansible

Также может понадобиться и некоторый другой софт, который описан в README.md каждой лабы. Также допускается
устанавливать софт в виртуалки и работать с ним там.
